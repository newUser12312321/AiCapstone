# 01. 프로젝트 구조 및 아키텍처

## 시스템 개요

라즈베리파이 5 **엣지**에서 웹캠으로 PCB를 촬영하고 YOLO로 정렬·결함 탐지를 수행한 뒤, 판정과 메타데이터를 **중앙 서버(Spring Boot + MySQL)** 로 전송합니다. 운영자는 **React(Vite)** 대시보드로 이력·통계를 봅니다. 현장 터치 단말에는 **Blazor WebAssembly** 키오스크(`frontend-kiosk`)를 사용할 수 있습니다.

## 데이터 흐름 (요약)

```
[Pi — edge :8000]
  캡처 → 추론 → GPIO 알람(선택)
      → POST /api/inspections (또는 /api/v1/inspections) → Spring Boot
[PC/서버 — backend :8080]
  MySQL 저장
      → GET 폴링/조회
[PC — frontend :5173]
  차트·테이블·이미지 오버레이 (/api, /edge 프록시)
```

## 포트 (기본값)

| 서비스 | 포트 | 위치 |
|--------|------|------|
| FastAPI (edge) | 8000 | 라즈베리파이(또는 Docker) |
| Spring Boot | 8080 | PC / Docker |
| Vite 개발 서버 | 5173 | PC / Docker |
| MySQL | 3306 (또는 compose에서 3307) | PC / Docker |

## REST 요약

### Spring Boot (`/api/inspections` 등)

검사 결과 수신·목록·단건·통계·기간 조회 등은 `backend`의 `InspectionController`를 기준으로 합니다. 일부 빌드에서는 **`/api/v1/inspections`** 호환 경로가 함께 제공될 수 있습니다(엣지 전송기 설정과 맞출 것).

### FastAPI (edge)

| Method | 경로 | 설명 |
|--------|------|------|
| GET | `/edge/health` | 헬스 |
| GET | `/edge/status` | 카메라·모델 상태 |
| POST | `/edge/inspect/trigger` | 수동 검사 |
| POST | `/edge/inspect/dummy` | 더미 전송 테스트 |
| GET | `/captures/{filename}` | 캡처 이미지 |

자세한 필드·재전송 큐 등은 [13_엣지_클라우드_전환_실행계획.md](13_엣지_클라우드_전환_실행계획.md)와 소스의 `edge/api`를 참고합니다.
