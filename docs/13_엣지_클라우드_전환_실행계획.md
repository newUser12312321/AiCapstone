# 엣지-클라우드 전환 실행계획 (확정본)

## 1) 확정 아키텍처

### Edge (Raspberry Pi)
- 카메라 스트림 처리
- 모델 추론/검사 로직 수행
- 판정 결과 즉시 표시(터치 키오스크 UI)
- 네트워크 단절 시 로컬 큐(SQLite) 적재 후 재전송

### Cloud (GCP)
- Spring API + DB + React 대시보드
- 결과 영구 저장
- 상세 이력/조회
- 장비 상태 모니터링

### 통신
- Pi -> Cloud: 검사 결과 업로드 (1단계 REST, 2단계 MQTT 확장)
- Cloud -> Pi: 설정/모드 변경 (1단계 REST polling, 2단계 MQTT downlink)

## 2) 기능 분리 기준 (기존 코드 기준)

### Cloud에 남길 기능
- 사용자/권한
- 설비 목록/장비 메타 정보
- 결과 조회/검색/리포트
- 알람 이력
- 검사 이미지 상세 조회

### Pi로 내릴 기능
- 검사 시작/중지
- 카메라 제어
- 실시간 프리뷰
- 즉시 판정 UI (PASS/FAIL)

### 공통 계약 (고정)
- API version: `v1`
- 결과 스키마 핵심 필드:
  - `inspectionId`
  - `deviceId`
  - `timestamp(inspectedAt)`
  - `verdict(result)`
  - `defectType`
  - `confidence`
  - `imagePath`
- 이미지 확인 요구사항:
  - Pi가 원본 이미지를 Base64로 함께 업로드
  - Cloud가 파일로 저장 후 조회 URL(`imagePath`) 발급

## 3) 통신 방식 로드맵

### 1단계 (빠른 구축)
- HTTPS REST
- 업로드 엔드포인트: `POST /api/v1/inspections`
- Pi 로컬 재전송 큐: SQLite
  - 즉시 전송 실패 시 큐 적재
  - 이후 요청 시 큐 선재전송

### 2단계 (확장)
- MQTT 도입
- 실시간 상태/원격 제어 빈도가 높아질 때 전환

## 4) 이번 반영 사항

- Spring 컨트롤러에 `v1` 경로 추가:
  - `/api/v1/inspections` (기존 `/api/inspections` 호환 유지)
- 검사 이미지 업로드/저장 경로 반영:
  - 요청에 `imageBase64`, `imageMimeType` 지원
  - 서버 저장 후 `imagePath`를 `/api/v1/inspections/images/{filename}`로 저장
  - 이미지 조회 엔드포인트 추가
- Edge 전송기 개선:
  - 기본 업로드 경로를 `v1`로 고정
  - 로컬 SQLite 재전송 큐 추가
  - 전송 실패 payload 자동 큐 적재
  - 전송 시작 전 큐 선재전송
  - 옵션으로 이미지 Base64 동봉 전송

## 5) 운영 시 체크

- GCP 배포 시 `app.inspection-image-dir`을 영속 스토리지 경로로 지정
- 이미지 용량 증가 대비:
  - 보관 주기/압축/썸네일 정책 수립
- 장비 인증:
  - 디바이스 키/JWT를 추후 필수화
